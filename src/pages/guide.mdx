## 1. Create a Wordpress instance

[Log in to Saucer](/dashboard) using your GitHub account and click the **Create instance** button to configure a new Wordpress instance. Set your website title, and admin username and password. This will be the username/password combination that you'll use to log in to your Wordpress dashboard for the first time. You may also install some optional plugins and their GraphQL counterparts from this panel.

## 2. Bootstrap a Gatsby website

Once your Wordpress instance is ready to use, you can start consuming its data in a static Gatsby website with little effort. First, create your project directory and install a few NPM packages as dependencies:

```bash
mkdir my-project
cd my project
npm init -y # create a package.json
npm install gatsby react react-dom gatsby-source-graphql react-helmet gatsby-plugin-react-helmet # install dependencies
touch gatsby-config.js # create an empty config file
```

Next, create a new Gatsby config file and configure a new GraphQL source using your new Wordpress instance. Replace `YOUR-INSTANCE-NAME` with the unique name given to your instance:

```js
// gatsby-config.js
module.exports = {
  plugins: [
    'gatsby-plugin-react-helmet',
    {
      resolve: 'gatsby-source-graphql',
      options: {
        typeName: 'Wordpress',
        fieldName: 'wordpress',
        url: 'https://YOUR-INSTANCE-NAME.saucer.dev/graphql'
      }
    }
  ]
};
```

## 3. Create a homepage

Let's create a homepage for your website that reads from the newly configured GraphQL source. We'll create a React component at `src/pages/index.js` that renders the website title and description; and lists your blog posts. The file should end up looking something like this:

```jsx
// src/pages/index.js
import React, {Fragment} from 'react';
import {Helmet} from 'react-helmet';
import {Link, graphql} from 'gatsby';

export default function Home(props) {
  // page queries make their results available on the `data` prop
  const {generalSettings, posts} = props.data.wordpress;
  const {title, description} = generalSettings;

  return (
    <Fragment>
      {/* use react-helmet to add a page title and description */}
      <Helmet>
        <title>{title}</title>
        <meta name="description" content={description} />
      </Helmet>

      {/* render the title and description in the page */}
      <h1>{title}</h1>
      <p>{description}</p>

      {/* loop through posts and render their title and excerpt */}
      {posts.nodes.map(post => (
        <div key={post.id}>
          <h3 style={{color: 'red'}}>
            <Link to={`/posts/${post.slug}`}>{post.title}</Link>
          </h3>
          <h5>Posted on {new Date(post.date).toLocaleDateString()}</h5>
          <div dangerouslySetInnerHTML={{__html: post.excerpt}} />
        </div>
      ))}
    </Fragment>
  );
}

// use a page query to retrieve data from your GraphQL source
export const pageQuery = graphql`
  {
    wordpress {
      generalSettings {
        title
        description
      }
      posts {
        nodes {
          id
          slug
          date
          title
          excerpt
        }
      }
    }
  }
`;
```

## 4. Create post pages

Now that you have an index page, it's time to create pages for each post. To do this, we'll use the Gatsby `createPages` API in a `gatsby-node.js` file.

```js
// gatsby-node.js
exports.createPages = async ({actions, graphql}) => {
  const data = await graphql(`
    {
      wordpress {
        posts {
          nodes {
            id
            slug
          }
        }
      }
    }
  `);
  
  const postTemplate = require.resolve('./src/components/post-template.js');
  data.wordpress.posts.nodes.forEach(post => {
    actions.createPage({
      path: `/posts/${post.slug}`,
      component: postTemplate,
      context: {
        id: post.id
      }
    })
  });
};
```

```jsx
// src/components/post-template.js
import React, {Fragment} from 'react';
import {graphql} from 'gatsby';

export default function PostTemplate(props) {
  return (
    <Fragment>
      <h1>{props.data.wordpress.post.title}</h1>
    </Fragment>
  );
}

export const pageQuery = graphql`
  query PostQuery($id: String) {
    wordpress {
      post(id: $id) {
        title
      }
    }
  }
`;
```
